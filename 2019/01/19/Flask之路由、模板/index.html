Flask之路由、模板

1.什么是Flask：Flask是一个基于Python并且依赖于jinja2模板引擎和werkzeug WSGI

服务的一个微型框架，Flask中包含一个轻量级的Web服务器主要用于在开发阶段测试使用

​ Flask是基于MTV框架模式开发出来的框架

​ WSGI：Web Server Gateway Interface（Web服务网关接口）

​ 2.flask.version

5.路由

​ 1.什么是路由：

​ 处理url和函数之间的关系的程序就是”路由”

​ 请求地址到视图处理函数的映射

​ 2.路由的体现：在Flask中，路由是通过@app.route（）装饰器来表示的

​ ①路由的基本表示

​ #http：//127.0.0.1:5000/

​ @app.route(“/”)

​ def index();

​ return “xxx”

​ #http：//127.0.0.1:5000/adc/login

​ @app.route(“/adc/login”)

​ def index();

​ return “xxx”

​ ②带参数的路由：路由中可以携带参数来表示要传递到视图中的数据

​ #http：//127.0.0.1:5000/show/laowang

​ #http：//127.0.0.1:5000/show/wangwc

1.基本带参路由

@app.route(‘/show/’)

def show1(name):

​ #name:表示地址栏传递的数据

​ pass

​ 2.带多个参数的路由

​ 多个参数使用 / 隔开，并继续使用 <> 来表示

​ @app.route(‘/show1//‘)

​ def show(name,age)

​ pass

​ 3.指定参数类型的路由：允许在声明路由时指定参数

​ 类型避免了在程序中再转换

@app.route(‘/show//int:age’)

def show1(name,age):

​ name:字符串

​ age：整数

pass

​ int：类型转换器，

​ Flask中所支持的类型转换器：

类型转换器 作用
缺省 字符串，不能有斜杠（/）
int 整数
float 浮点型
path 字符串，可以存在斜杠（/）

​ 4.多url的路由匹配

​ 为多个URl访问地址匹配同一个视图函数

@app.route(‘/地址1’)

@app.route(‘/地址2’)

@app.route(‘/地址3’)

def show1():

pass

端口占用的问题解决方案

1 终端中netstat –lptu 查找占用5000端口的pid

Sudo kill -9 进程ID 将pid杀死

2 重新指定端口

​ 3.路由中设置HTTP请求方法

​ Flask中默认只能接受get请求，无法接受post请求，

​ 在Flask中允许设定可以接收的请求方式，如果请求

​ 方式不匹配的话，会响应回405（method Not Allowed）

​ @app.route（”/xxx”,methods=[‘POST’,’GET’]）

​ def xxx();

​ 该函数既能接收post请求也能接收get请求

​ 4.URL的反向解析

​ 正向解析：程序自动解析，根据@app.route（）中的访问路径，来匹配处理函数

​ 反向解析：通过视图处理函数的名称自动生成对应的访问路径

​ 在Flask中要实现反向解析:url_for(funname，args)

​ funName：要生成地址的函数名

​ args：该地址中需要的参数
